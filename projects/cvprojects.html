Computer Vision Projects

Projects From Dr. Bryan Klingner's Computer Vision and 3D Reconstruction Class


Class Overview

In this lab-based class, we dove into practical applications of 3D reconstruction, combining hardware and software to build our own 3D environments from scratch. We used open-source framework, OpenCV, to do the heavy lifting, with the focus on understanding and applying state-of-the art approaches to geometric computer vision.

Panorama Stitching

For this project, we wrote software to stitch multiple images of a scene together into a panorama automatically.

The first step for the panorama stitcher is finding a homography between the two images. To establish a homography between two images, we first needed to find a set of correspondences between them. The way I chose to do this was to identify "interest points" or "key points" in both images, summarize their appearances using descriptors, and then establish matches between these "features" (interest points combined with their descriptors) by choosing features with similar descriptors from each image using the cv2.SIFT interface to compute interest points and descriptors.

Once correspondences were established between two images, I used them to find a "best" homography mapping one image into the frame of another using cv2.findHomography, which computes the least-squares-best-fit homography given a set of corresponding points. It also offers robustified computation using RANSAC or the Least-Median methods.

Once homographies were found, I warped them according to these homographies in order to composite them together using cv.warpPerspective.

Stereo Imaging

In this project, we reconstructed a 3D scene from stereo image pairs.

Just like for panorama stitching, the first thing I needed to do was to establish some correspondences between the images in order to compute epipolar geometry from a pair of images. Next, I got the fundamental matrix using cv2.findFundamentalMat(). Next, I rectified the images using cv2.stereoRectifyUncalibrated().

With the rectified stereo pair, I then computed disparities between correponding pixels using StereoSGBM. I then used disparity and camera focal length to find depth to get results in 3D. I then converted the disparity images to a PLY point cloud.

Object Tracking

In this project, we tracked objects in video sequences using background subtraction, optical flow, sparse feature tracking, and Kalman filtering.

We first tracked a ball on a checkered background and then a face moving through a scene without using any ground truth to make the facial tracking easier.
