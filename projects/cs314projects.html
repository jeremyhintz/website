Data Structures

Projects for CS 314: Data Structures


Class Overview

CS 314 covers methods for storing large amounts of data in data structures, the algorithms used to efficiently access and manipulate that data, and analysis of the performance of the algorithms. Only the interesting projects are described here for brevity.

Trees and Symbolic Algebra

In this project, we wrote a recursive function solve(Cons e, String v) that attempted to solve an equation e for variable v. We assumed that any variable occured at most once in e, and that the equation initially had a left-hand side that was a single variable. The basic idea of solve was to test for equations that were already solved or unsolvable (base cases); otherwise, solve would search for a solution by applying laws of algebra to transform the equation into different forms. solve() is a kind of tree recursion -- not recursion on first and rest, but recursion on the applicable transformations of the input formula into new formulas. We were given the following physics functions to start with:

  • (= s (* 0.5 (* a (exp t 2))))
  • (= s (+ s0 (* v t)))
  • (= a (/ f m))
  • (= v (* a t))
  • (= f (/ (* m v) t))
  • (= f (/ (* m (exp v 2)) r))
  • (= h (- h0 (* 4.94 (exp t 2))))
  • (= c (sqrt (+ (exp a 2) (exp b 2))))
  • (= v (* v0 (- 1 (exp (/ (- t) (* r c))))))

We were then given physics problems and asked to write formulas that would solve them.

Memos and Music

In this project, we were asked to write software for a synthesizer that would play songs by using a few commands that could be strung together in a tree pattern and played back using JFugue.

  • (rest d) does not emit anything, but has a time duration of d.
  • (boom d) emits a bass_drum command.
  • (snare d) emits a snare drum command.
  • (hat d) emits a high-hat cymbal command.
  • (cymbal d) emits a crash cymbal command.
  • (cowbell d) emits a cowbell command.
  • (bell d) emits a ride bell command.
  • (clap d) emits a hand clap command.
  • (tambourine d) multiplies its duration by a random number, obtained by calling random.nextDouble(); if the integer result is > 2, it schedules tambourine again with that amount as its duration, at the current time plus current duration minus that amount. It emits a command with a duration of 2 immediately. (This random behavior simulates a tambourine player who smokes way too much weed).
  • (seq action1 ... actionn) schedules each of its actions to execute in sequence, one after the other. seq thus makes a sequence of multiple actions look like a single action. seq could either schedule all of its actions at once, starting with the current time and adding the time of each action, or it could schedule the first action and schedule a seq for the rest of the actions.
  • (sync action1 ... actionn) schedules each of its actions for the current time. sync represents synchronized execution of multiple subprograms.
  • (repeat n action) schedules n events with the specified action. The first of these n events will happen at the same time that repeat executes (i.e. simultaneously with repeat). One way to accomplish this is to schedule the action immediately; then if n > 1, schedule (repeat n-1 action) for the current time plus the time of the action. Alternatively, repeat could simply schedule all n events at once; this would use more queue space if n is large.
  • (piano voice note d) emits a piano command using the function emitp(voice, note, time, d). A note is a string such as "C". A note may specify an octave, such as "C5", and the note letter may be followed by "#" or "b" for sharp and flat. voice is an integer from 0 to 3.

Do You Know the Way to San Jose Muleshoe?

Even Dijkstra's algorithm is bigger in Texas. In this assignment, we looked at graph algorithms like Dijkstra's algorithm, Prim's algorithm, and A* search. We applied them to map traversal like what is done on Google Maps, where the nodes are the cities of Texas and the edges are highways with a weight given by their relative distances.

MapReduce

In this assignment, we learned the basics of MapReduce programming. MapReduce is used by Google for much of their processing of large data sets (terabytes of data). Hadoop is an open-source version of MapReduce written in Java; it is available to commercial users through Amazon Web Services. For this assignment, we used a simplified version of MapReduce; the way in which it is programmed is similar to Hadoop.
